## 🎯 Review 目标

本次 Code Review 重点关注：

* 代码质量（可读性、可维护性）
* Vue3 组合式 API 使用是否合理
* 状态管理（Pinia）是否规范
* uni-app 多端兼容问题
* 是否存在潜在 bug / 性能问题

---

## 🧩 Review 维度（必须逐项分析）

### 1️⃣ 代码结构 & 可维护性

重点检查：

* 是否存在重复逻辑（是否可抽离 composable / utils）
* setup 是否过于臃肿（是否应拆分）
* 命名是否清晰（变量 / 方法 / store）
* 是否存在魔法值（建议抽常量）

---

### 2️⃣ Vue3（组合式 API）

重点检查：

* ref / reactive 使用是否合理
* 是否滥用 reactive（推荐优先 ref）
* watch / computed 是否合理
* 是否存在副作用未清理问题

---

### 3️⃣ Pinia 状态管理

重点检查：

* 是否滥用全局状态（本地状态更合适？）
* state 是否冗余
* action 是否职责清晰
* 是否存在跨页面污染风险

---

### 4️⃣ uni-app 多端问题

重点检查：

* 是否使用不兼容 API
* 是否考虑 H5 / 小程序差异
* 样式是否可能存在端差异问题

---

### 5️⃣ 功能逻辑 & Bug 风险

重点检查：

* 是否存在明显 bug（空值、未定义）
* 是否缺少异常处理（try/catch）
* 是否考虑用户极端操作（重复点击等）
* 接口失败是否处理

---

### 6️⃣ 性能优化

重点检查：

* 是否存在重复请求
* computed / watch 是否滥用
* 是否有不必要的渲染
* 大数据量是否有优化空间

---

### 7️⃣ 用户体验（UX）

重点检查：

* 是否有 loading 状态
* 是否有错误提示
* 是否有空数据处理
* 是否存在卡顿风险

---

## 📊 输出格式（必须严格遵守）

---

### 🧾 一、整体评价

* 代码质量评分：**X / 10**
* 风险等级：**高 / 中 / 低**
* 总结一句话评价

---

### ❌ 二、问题（必须修改）

> 只列出严重问题（bug / 架构问题 / 明显不合理）

格式：

```md
❌ 问题1：xxx

- 问题说明：
- 影响范围：
- 修改建议：
- 优化示例代码：
```

---

### ⚠️ 三、优化建议（推荐）

```md
⚠️ 建议1：xxx

- 原因：
- 建议：
- 示例代码（如有）：
```

---

### ✅ 四、做得好的地方

```md
✅ xxx 写法合理：原因说明
```

---

### 🧠 五、重构建议（加分项）

* 如果这段代码“可以明显优化结构”，请给出：

  * 重构思路
  * 优化后的代码结构（可简化版）

---

## ⚠️ 强制要求（非常重要）

1. ❗ 不要只说问题，必须给“修改方案”
2. ❗ 能给代码示例就必须给
3. ❗ 不要泛泛而谈（必须结合代码）
4. ❗ 优先指出最严重问题（最多 5 个）
5. ❗ 输出要简洁但专业（不要废话）
